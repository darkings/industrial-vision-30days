## 四、怎么学习

每天按照固定节奏学习，不要只看代码，也不要只看理论。

### 1. 每天开始前

```text
1. 打开 README.md，看当前下一课。
2. 打开 LEARNING_PROGRESS.md，看上一节做到哪里。
3. 打开 LEARNING_ERRORS.md，复习最近 3 个相关错误。
4. 打开当天 Obsidian 笔记。
```

### 2. 每一节课的学习顺序

```text
1. 先说工业问题：这节解决现场什么问题。
2. 再讲核心概念：比如曝光、阈值、轮廓、标定、置信度。
3. 再讲 Python / OpenCV / SDK API：每个参数是什么意思。
4. 看完整示范代码。
5. 逐段解释代码。
6. 学员跟写一次。
7. 修改 1-3 个参数观察结果。
8. 做一个小独立练习。
9. 检查输出图片、打印数据和理论回答。
10. 把结果同步到笔记和进度文件。
```

### 3. 每天结束前

```text
1. 至少保存一张原图、一张中间结果图、一张最终结果图。
2. 记录关键参数：曝光、增益、阈值、面积阈值、光源条件。
3. 记录今天最容易错的 1-3 个点。
4. 如果出现有价值错误，写入 LEARNING_ERRORS.md。
5. 更新 README.md 和 LEARNING_PROGRESS.md。
```

### 4. 当前阶段的学习重点

```text
现在不要急着 YOLO。
先把真实相机图像 -> OpenCV 阈值 -> 轮廓 -> 报告图 这条链路打稳。
支架、光源、控制器到位后，先学习“稳定成像系统”。
先固定画面和光照，再选曝光和阈值，最后才谈检测规则。
控制器先学习作用、接线和触发逻辑，不急着一开始就做复杂联动。
```
