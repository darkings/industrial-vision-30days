## 教学代码注释规范

以后所有用于教学的代码必须包含详细中文注释。

注释至少需要说明：

- 导入的包或模块有什么作用。
- 变量保存的是什么数据。
- 函数解决什么问题。
- OpenCV API 的每个参数表示什么。
- 图片数据的 shape 和通道发生了什么变化。
- 坐标使用 `(x, y)` 还是 `[y, x]`。
- 为什么要判断 `None`。
- 输入文件来自哪里，输出文件保存到哪里。
- 容易写错或容易混淆的地方。

示例：

```python
# cv2.rectangle() 会直接修改传入的图片。
# 因此先复制原图，避免后续仍然需要无标记的原始图片时数据已经被改变。
marked = image.copy()

# 在 marked 图片上绘制绿色矩形框。
cv2.rectangle(
    marked,          # 要绘制的图片，是一个 NumPy 数组
    (x1, y1),        # 矩形左上角坐标，顺序是 (x, y)
    (x2, y2),        # 矩形右下角坐标，顺序是 (x, y)
    (0, 255, 0),     # BGR 颜色：蓝0、绿255、红0，因此是绿色
    2                # 线条粗细，单位是像素
)
```

详细注释用于学习阶段。等知识掌握后，正式项目代码会减少重复注释，只保留解释设计原因和复杂逻辑的注释。
